# Personalized Wardrobe Assistant

An AI-powered fashion assistant that understands your wardrobe through computer vision, recommends outfits via vector similarity search, and generates photorealistic virtual try-ons using generative AI.

---

## Features

- **Smart wardrobe upload** — Upload individual clothing photos or a `.zip` of your whole closet. Items are auto-segmented (Grounded SAM), embedded (FashionCLIP), and categorized (Gemini Vision).
- **Outfit suggestions** — FAISS-powered similarity search pairs tops and bottoms from a reference outfit corpus. Falls back to HSL color-harmony pairing when the library is unavailable.
- **Virtual try-on** — Select a body photo + outfit and get a photorealistic image of yourself wearing it, generated by Gemini 2.5 Flash.
- **Similar item lookup** — Click any wardrobe item to find visually similar pieces from the outfit library.
- **User authentication** — JWT-based auth with bcrypt password hashing and login history tracking.

## Setup

### Prerequisites

- Python 3.10+
- Node.js 18+
- Docker (for PostgreSQL)
- A [Google Gemini API key](https://aistudio.google.com/app/apikey)

### 1. Clone & configure environment

```bash
git clone https://github.com/your-username/Personalized-Wardrobe-Assistant.git
cd Personalized-Wardrobe-Assistant
```

Copy and fill in both env files:

```bash
cp .env.example .env
cp backend/.env.example backend/.env
```

**Root `.env`:**
```
OUTFIT_FOLDER_URL=https://drive.google.com/drive/folders/YOUR_FOLDER_ID_HERE?usp=drive_link
```

**`backend/.env`:**
```
GEMINI_API_KEY=your_gemini_api_key
DATABASE_URL=postgresql+psycopg2://wardrobe:wardrobe@localhost:5432/wardrobe
GEMINI_IMAGE_MODEL=gemini-2.5-flash-preview-05-20
GEMINI_VISION_MODEL=gemini-2.5-flash
```

### 2. Start PostgreSQL

```bash
docker compose up -d
```

### 3. Install backend dependencies

```bash
pip install -r requirements.txt
pip install -r backend/requirements.txt
```

### 4. Build the outfit library (optional but recommended)

This downloads outfit images from your Google Drive folder, runs SAM + FashionCLIP, and writes the FAISS index:

```bash
python scripts/build_library.py
```

> Skip this step to run in wardrobe-only mode (outfit suggestions will use color pairing instead of FAISS retrieval).

### 5. Start the backend

```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

### 6. Start the frontend

```bash
cd frontend
npm install
npm start
```

The app will be available at [http://localhost:3000](http://localhost:3000).

## API Overview

All endpoints are under `http://localhost:8000/api/`.

| Method | Path | Description |
|---|---|---|
| `POST` | `/auth/signup` | Register a new user → JWT |
| `POST` | `/auth/login` | Login → JWT |
| `GET` | `/auth/me` | Get current user profile |
| `POST` | `/wardrobe/items` | Upload a clothing item image |
| `POST` | `/wardrobe/items/bulk` | Upload a `.zip` of clothing images |
| `GET` | `/wardrobe/items` | List all wardrobe items |
| `GET` | `/wardrobe/items/{id}/similar` | Find similar items via FAISS |
| `DELETE` | `/wardrobe/items/{id}` | Delete a wardrobe item |
| `POST` | `/wardrobe/body-photo` | Upload full-body photo |
| `GET` | `/outfits/suggestions` | Get outfit suggestions |
| `POST` | `/tryon/generate` | Generate virtual try-on image |
| `POST` | `/health/warm` | Warm up FashionCLIP + FAISS (called at boot) |

Static files are served at `/uploads/`, `/library/`, and `/crops/`.

## How It Works

### Wardrobe Ingestion

1. Uploaded image is saved to `uploads/`.
2. Grounded SAM detects and segments the garment → transparent-background crop saved to `uploads/crops/`.
3. FashionCLIP encodes the crop into a 512-dim L2-normalized embedding.
4. Gemini Vision categorizes the item (Top / Bottom / Shoes / Outerwear / Accessory).
5. k-means clustering extracts a dominant color palette (HSL).
6. Item metadata is stored (currently in-memory; PostgreSQL models ready to swap in).

### Outfit Suggestions

- **FAISS mode**: Each user item is queried against the pre-built category FAISS index. Results are grouped by outfit ID and ranked by average similarity score. Top 12 pairs are returned.
- **Color-harmony fallback**: If the library index is unavailable, tops and bottoms in the user's wardrobe are paired by minimizing HSL color distance.

### Virtual Try-On

The frontend sends three images (body photo, top crop, bottom crop) as a multipart form to `/api/tryon/generate`. The backend calls Gemini 2.5 Flash image generation with a prompt instructing it to dress the person in the selected outfit while preserving pose, face, and background. The result streams back as a PNG/JPEG.

## Development Notes

- **Auth is in-memory** by default (`backend/app/core/store.py`). SQLAlchemy models and Alembic migrations are present in `backend/app/models/` and `backend/migrations/` — swap the router calls to use the DB session to enable persistence.
- **Model cold-start**: FashionCLIP and FAISS take ~10–30 seconds to load. The frontend calls `/api/health/warm` at boot to pre-load them before the first user action.
- **Demo account**: The login page has a "Use demo account" button that auto-creates a test user — useful for quick demos.
